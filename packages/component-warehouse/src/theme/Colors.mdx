# Color Theme System

The APAC UI Warehouse uses a standardized color palette that ensures consistency across the entire application. All colors are available in both SCSS variables and TypeScript constants.

## Overview

Our color system is organized into several categories:

- **Primary Colors**: Main brand colors used for primary actions and highlights
- **Secondary Colors**: Supporting brand colors for secondary actions
- **Semantic Colors**: Success, Warning, Error, and Info states
- **Text Colors**: Colors for different text hierarchy levels
- **Background Colors**: Colors for backgrounds and surfaces
- **Border Colors**: Colors for borders and dividers
- **Grayscale**: Neutral gray scale from 50 to 900
- **Shadows**: Elevation and depth styles

## Getting Started

### Installation

The color theme is automatically included when you import the component warehouse:

```typescript
import { colors } from '@apac-ui-warehouse/component-warehouse';
```

### SCSS Usage

Import the colors in your SCSS files:

```scss
@import '@apac-ui-warehouse/component-warehouse/theme/colors.scss';

.my-component {
  background-color: $color-primary;
  color: $color-text-inverse;
  border: 1px solid $color-border;
  box-shadow: $shadow-md;
}
```

### TypeScript/JavaScript Usage

Use the colors object directly:

```typescript
import { colors, getColor } from '@apac-ui-warehouse/component-warehouse';

// Direct access
const primaryColor = colors.primary;

// Dynamic access with helper
const secondaryColor = getColor('secondary');

// Use in inline styles
const style = {
  backgroundColor: colors.primary,
  color: colors.textInverse,
  border: `1px solid ${colors.border}`,
};
```

## Best Practices

### 1. Consistency

Always use theme colors instead of hardcoded hex values. This ensures consistency and makes theme updates easier.

❌ **Don't:**

```scss
.button {
  background-color: #4e738a; // Hardcoded
}
```

✅ **Do:**

```scss
.button {
  background-color: $color-primary; // Theme color
}
```

### 2. Semantic Meaning

Use semantic colors appropriately. Don't use error colors for non-error states, or success colors for warnings.

### 3. Contrast

Ensure sufficient contrast between text and background colors for accessibility:

- Use `textPrimary` on light backgrounds
- Use `textInverse` on dark backgrounds
- Check contrast ratios meet WCAG AA standards

### 4. Hierarchy

Use text color hierarchy to guide user attention:

- Primary text for important content
- Secondary text for supporting information
- Tertiary text for less important details

### 5. States

Use hover, active, and disabled variants appropriately:

- Hover states should be slightly darker/lighter
- Disabled states should use `textDisabled` and `bgDisabled`
- Active states should clearly indicate selection

## Migration Guide

If you're migrating from hardcoded colors:

1. **Identify hardcoded colors** in your codebase
2. **Map to theme colors** using the color palette above
3. **Replace in SCSS**: Change `#hex` to `$color-*` variables
4. **Replace in TypeScript**: Change `'#hex'` to `colors.*`
5. **Test thoroughly** to ensure visual consistency

## Accessibility

All colors in the theme are designed with accessibility in mind:

- Text colors meet WCAG contrast requirements
- Semantic colors are distinguishable for colorblind users
- Dark mode support with `bgDark` and `bgDarkSecondary`

## Customization

To customize the theme for your brand:

1. Update values in `src/theme/colors.scss`
2. Update values in `src/theme/colors.ts`
3. Rebuild the component warehouse
4. Import the updated theme in your application
